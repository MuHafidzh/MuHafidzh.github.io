# Engineering Portfolio Master Plan

> **Vision**
>
> Build a portfolio that feels like an engineering product, not a personal portfolio template.
>
> The visitor should think:
>
> **"This person builds real embedded and autonomous systems."**

---

# Core Principles

The website is **not** a CV.

The CV summarizes experience.

The website tells the engineering story behind each project.

Every section should answer one question:

> **Can this person build real systems?**

---

# Design Direction

Inspired by

- Apple Developer
- NVIDIA Research
- Pony.ai
- Tier IV
- MIT CSAIL
- CMU Robotics
- Formula Student team websites

Avoid inspiration from:

- Generic portfolio templates
- Bootstrap portfolios
- Dribbble portfolios
- AI-generated portfolio layouts

---

# Visual Identity

Minimal.

Professional.

Engineering-first.

No unnecessary decorations.

No colorful icons.

No huge gradients.

No glassmorphism.

No floating particles.

The projects themselves should become the visual identity.

# Long-Term Vision

The website should eventually become a complete engineering portfolio.

Someone interested in hiring should be able to understand:

- what problems were solved
- how the systems were designed
- what technologies were used
- what engineering decisions were made

without opening the CV.

---

# Final Impression

The portfolio should not feel like:

"I am looking for a job."

Instead it should feel like:

"Here is the engineering work I have built over the past few years."

The projects become the proof.

The CV becomes only the summary.
